# Deploy MonoLit to Render + Neon (free, no credit card)

This guide deploys MonoLit on Render's free web service and uses Neon's free
serverless Postgres. Total time: ~10 minutes.

## 1. Create the Postgres database on Neon

1. Go to https://console.neon.tech and sign up (GitHub or email — no card).
2. Create a project. Pick a region close to your users (e.g. EU Central).
3. Open the project, go to **Dashboard → Connection Details**.
4. Copy the **pooled** connection string. It looks like:

   ```
   postgres://USER:PASSWORD@ep-xxx.eu-central-1.aws.neon.tech/neondb?sslmode=require
   ```

   Save it — you'll paste it into Render in the next step.

## 2. Deploy the web service on Render

1. Go to https://render.com and sign up (GitHub or email — no card).
2. Click **New → Web Service**.
3. Connect this GitHub repository.
4. Render will detect `render.yaml` and pre-fill most fields. Confirm:
   - **Runtime:** Python
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
   - **Plan:** Free
5. Under **Environment Variables**, set:
   - `DATABASE_URL` → paste the Neon connection string from step 1
   - `MONOLIT_SECRET` → leave blank (auto-generated by Render via `generateValue: true`)
   - `PYTHON_VERSION` → `3.11.10` (already in render.yaml)
6. Click **Create Web Service**. First build takes 3-5 minutes.

## 3. Verify it works

Visit your `https://<service-name>.onrender.com` URL.

- Public site loads.
- Register a new account → confirms Postgres write works.
- Log in with the seeded Owner account: **username** `pnexn` / **password** `MonoLitOwner!2025`.
- Change the Owner password immediately from the **Account** dropdown.

## Notes / quirks

- **Cold starts:** Render's free tier puts the service to sleep after 15 minutes
  of inactivity. The first request after sleep takes ~30 seconds while the
  container spins back up. Users in the database are NOT affected — they live
  on Neon, which is always-on and serverless.
- **Neon auto-suspend:** Neon also suspends inactive databases after a few
  minutes, but it wakes up in <1 second on the first query — invisible.
- **JWT secret:** `MONOLIT_SECRET` is generated by Render once and persists
  across redeploys. All cookies remain valid as long as the service exists.
- **Owner credentials:** changing them in Render env vars
  (`MONOLIT_OWNER_USERNAME` / `MONOLIT_OWNER_PASSWORD`) only affects what's
  seeded on first run. After that, change the password from the UI.
